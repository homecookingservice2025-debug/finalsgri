# 📘 Shri Krishna Management System - Universal User Manual

Welcome to the **Shri Krishna Management System**, a highly secure, integrated dual-module platform with CRM, advanced WhatsApp automation, granular administrative logging, and interactive analytical reporting custom-tailored for **Shri Krishna Mission Hospital** and **Shri Krishna Sugar & Dairy Co.**

This manual is divided into clear functional sections to guide Admins and Staff members in leveraging the system's full capabilities.

---

## 🧭 1. System Overview & Dual-Module Architecture

The system features an innovative **Dual-Module Mode Switch** that dynamically adjusts the entire application's views, fields, and operations to target either Hospital management or Dairy management:

1. **🏥 Hospital Module (Shri Krishna Mission Hospital):**
   * Designed for patient tracking, doctor assignments, department logs, and health status tracking.
   * Specific elements include **Aadhaar Verification**, **Doctor In Charge**, **Health Status (Active/Recovered/Under Observation)**, and custom **Prescription Document Generation**.

2. **🐄 Dairy Module (Shri Krishna Sugar & Dairy Co.):**
   * Tailored for milk procurement, supplier lists, dairy logistics, and sugarcane pricing updates.
   * Specific elements include **Milk Collection Metrics (Quantity in Liters, Fat Content %)**, **Verification Status**, and **Sugarcane Logistics Fields**.

*How to switch:* At the left navigation sidebar pane bottom, find the active module widget toggle (indicated by a 🏥 or 🚜 icon). Clicking **"Switch Module"** instantly migrates the navigation panels, search rules, analytics dashboards, and active templates.

## 👥 2. Administrative and Staff Access Roles

The security architecture implements role-based boundaries to ensure data protection and audit logging:

* **👑 System Administrator (Admin):**
  * full privileges to add/edit/delete patient or farmer records.
  * Define and edit Messaging Templates and Automation settings.
  * Create, edit, and delete Staff member accounts.
  * Execute bulk imports and data clearing operations.
  * Manipulate Master geographic tables (States, Districts, Blocks, Posts).

* **🩺 Staff Member (Staff):**
  * Read-only and creation permission on entries depending on specific workflows.
  * Send custom WhatsApp greetings and documents to individuals.
  * Read log catalogs and view interactive data dashboards.
  * Restricted from destroying data, manipulating other staff accounts, or modifying global API credentials.

*How to Authenticate:* Use the login portal using either the developer `admin` password configuration or registered Staff credentials generated by the administrator.

## 📊 3. Core Data Management & Geography Master Validation

To ensure extreme integrity when capturing patient or farmer contact information, the system enforces a strict geographic structure:
$$\text{State} \longrightarrow \text{District} \longrightarrow \text{Block} \longrightarrow \text{Post Office (with Pincode)} \longrightarrow \text{Village}$$

### Creating and Validating Geography Masters:
1. Navigate to the **Masters Drawer** in the left sidebar (Admin role required).
2. Populate the tables progressively:
   * **State Master:** Add relevant States (e.g., *Uttar Pradesh*).
   * **District Master:** Link new Districts to active States (e.g., *Basti*).
   * **Block Master:** Associate Blocks with active Districts (e.g., *Bargodwa*).
   * **Post Office Master:** Map Post names and enter precise 6-digit Pincodes (e.g., *272001*).
3. **Automatic Form Insertion:** When adding a new Patient or Farmer, select the geographic nodes from the reactive dropdown fields. This auto-populates the correct Pincode, avoiding typographic errors and verifying compliance.

## 💬 4. Dynamic Messaging Templates with Placeholder Substitution

Create pre-written structures for your campaigns to standardize outreach:

### Standard Dynamic Placeholders:
Use double curly-braces to inject client variables dynamically at run-time:
* `{{name}}` $\longrightarrow$ Substitutes the full name of the Patient or Farmer.
* `{{id}}` $\longrightarrow$ Substitutes the registered account ID.
* `{{village}}` $\longrightarrow$ Substitutes the recorded village name.
* `{{doctor}}` $\longrightarrow$ Substitutes the assigned doctor's name (Hospital).
* `{{quantity}}` $\longrightarrow$ Substitutes the registered daily milk collection capacity (Dairy).

### How to Create and Apply Templates:
1. Click **Messaging Automation** in the sidebar.
2. Scroll to the **Templates** section and click **Create Template**.
3. Choose a template type category: **Birthday Greeting**, **Anniversary Greeting**, **Bulk Outreach**, or **Custom**.
4. Test the variables! When launching a message, selecting the custom template will automatically format the message body for each individual contact.

## 📢 5. High-Impact WhatsApp Campaigns & Broadcaster Setup

Run automatic multi-contact campaigns with custom delayed scheduling to prevent accounts from being flagged or rate-limited:

1. **Connect Multi-Device Account:**
   * Go to **Advanced WhatsApp CRM**.
   * Click **Add WhatsApp Account**, assign an account label, and click **Link Device**.
   * Run the manual linking or wait for the QR code display to pair the container backend engine using standard multi-device pairing.

2. **Build and Queue a Campaign:**
   * Under the **Messaging Automation** workspace, click **Launch Campaign**.
   * Input the Campaign Name and select the connected sender WhatsApp profile.
   * Draft the Message Content or choose an active **Message Template** from the registry.
   * Drag & drop a contact sheet or manually paste target recipient list records.
   * **Set Safe Interval Delays:** Input your preferred safety spacing factor (e.g., `5-10 seconds` per outbound dispatch).
   * Click **Broadcast Campaign**. You will observe real-time metrics tracking total sent, pending queue length, and failure percentages.

## 🧾 6. Digital Docs & Smart Invoices

Send customized receipts and bills to your users directly on their WhatsApp line:

1. Open **Advanced WhatsApp CRM** and scroll to the **Smart Docs & Invoices** panel.
2. Click **Create Custom Invoice / Receipt**.
3. Input client information: **Recipient Name**, **WhatsApp Number**, **Transaction Code/Reference**, and **Total Charges/Amount**.
4. Detail purchased assets or health services under the **Items/Consultation Details** text form.
5. Choose the dispatch formatting template.
6. Click **Generate & Send Document**. The system formats a stunning, readable billing brief message along with PDF attachment simulation and routes it directly to the designated mobile device.

## 🛡️ 9. Offline Reliability Mode & Fallback Databases

What happens if the primary cloud database configuration displays a lockout state?
The app is built with a state-of-the-art **dual-layer protection engine**:

* **🔄 Autonomic Fallback Detection:**
  * When database structures are inaccessible, the system immediately diverts read/write paths to local, high-speed memory buffers.
  * Patient, Farmer, and Master entries are saved into `/src/initial_patients.json` and `/src/initial_dairy.json` natively on the application volume.
  * This ensures no operator is ever blocked from creating patient entries, defining message templates, or registering logs even in remote network circumstances. When connection restores, fallback data is ready to sync perfectly!
